Crop Migration
GitHub → Firestore
Reads all .js crop files from the crops/ folder in your GitHub repo, parses each Crop({…}) call, and writes the data to the crops collection in Firestore. Run this whenever you add or update crop files.
Options
Prune stale documents
Delete Firestore docs whose IDs no longer exist in GitHub. Use after renaming or removing crop files.
Actions
What this writes
Each crop becomes a document at crops/{id} (e.g. crops/tomatoes). The document contains all crop fields plus a _migratedAt timestamp.

After migrating
Replace the GitHub-based loadAllCrops() in index.html and loadCrops() in garden.html with the Firestore loader snippet shown at the bottom of this page.

Drop-in Firestore loader for index.html & garden.html
// ── Add this import at the top of your <script type="module"> ──
import { getFirestore, getDocs, collection }
  from "https://www.gstatic.com/firebasejs/10.12.2/firebase-firestore.js";

// ── Replace loadAllCrops() / loadCrops() with this ──
async function loadAllCrops() {
  const db   = getFirestore();   // already initialised via initializeApp()
  const snap = await getDocs(collection(db, 'crops'));
  PLANTS = [];
  snap.forEach(d => {
    const data = d.data();
    // Normalise companions (same as before)
    const norm = arr => (arr || []).map(x =>
      typeof x === 'string' ? { name: x, why: '' } : x
    );
    data.companions = {
      good: norm(data.companions?.good),
      bad:  norm(data.companions?.bad),
    };
    PLANTS.push({ data, sha: null, filename: d.id + '.js', path: null });
  });
  PLANTS.sort((a, b) => a.data.name?.localeCompare(b.data.name));
}
